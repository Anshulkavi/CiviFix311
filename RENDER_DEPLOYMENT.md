# CivicFix - Render Deployment Guide

Deploy CivicFix to Render in minutes using the automated blueprint.

## Prerequisites

- GitHub account
- Render account (free tier available at render.com)
- Git repository with your code

---

## Quick Deploy (Recommended)

### 1. Push to GitHub

```bash
# Initialize git if not already done
git init
git add .
git commit -m "Initial commit"

# Add remote and push
git remote add origin https://github.com/YOUR_USERNAME/civicfix.git
git branch -M main
git push -u origin main
```

### 2. Connect to Render

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click **"New"** → **"Blueprint"**
3. Connect your GitHub repository
4. Select the `civicfix` repository
5. Render will automatically detect `render.yaml`

### 3. Configure Environment Variables

Before deployment, set these required environment variables in Render:

#### Backend Service Variables:
- `ALLOWED_HOSTS` - Your backend URL (e.g., `civicfix-backend.onrender.com`)
- `CORS_ALLOWED_ORIGINS` - Your frontend URL (e.g., `https://civicfix-frontend.onrender.com`)
- `EMAIL_HOST_USER` - (Optional) Your SMTP email
- `EMAIL_HOST_PASSWORD` - (Optional) Your SMTP password

**Note:** `SECRET_KEY` and `DATABASE_URL` are auto-generated by Render.

#### Frontend Service Variables:
- `REACT_APP_API_URL` - Your backend URL (e.g., `https://civicfix-backend.onrender.com/api`)

### 4. Deploy

Click **"Apply"** and Render will:
- Create PostgreSQL database
- Build and deploy backend
- Build and deploy frontend
- Run migrations and seed demo data

## Manual Deployment (Alternative)

If you prefer manual setup:

### Database

1. **Create PostgreSQL Database**
   - Dashboard → New → PostgreSQL
   - Name: `civicfix-db`
   - Plan: Free
   - Copy the **Internal Database URL**

### Backend

1. **Create Web Service**
   - Dashboard → New → Web Service
   - Connect repository → Select branch `main`
   - Name: `civicfix-backend`
   - Runtime: Python 3
   - Build Command: `pip install -r backend/requirements.txt && python backend/manage.py collectstatic --no-input`
   - Start Command: `cd backend && gunicorn civicfix.wsgi:application`
   - Plan: Free

2. **Environment Variables:**
   ```
   PYTHON_VERSION=3.12.0
   SECRET_KEY=<generate-random-50-chars>
   DEBUG=False
   DATABASE_URL=<from-step-1>
   ALLOWED_HOSTS=<your-backend-url>.onrender.com
   CORS_ALLOWED_ORIGINS=https://<your-frontend-url>.onrender.com
   ```

3. **Advanced Settings:**
   - Auto-Deploy: Yes
   - Health Check Path: `/api/health/`

### Frontend

1. **Create Static Site**
   - Dashboard → New → Static Site
   - Connect repository
   - Name: `civicfix-frontend`
   - Build Command: `cd frontend && npm install && npm run build`
   - Publish Directory: `frontend/build`

2. **Environment Variables:**
   ```
   REACT_APP_API_URL=https://<your-backend-url>.onrender.com/api
   ```

3. **Redirects/Rewrites:**
   - Source: `/*`
   - Destination: `/index.html`
   - Action: Rewrite

## Post-Deployment

### 1. Create Admin User

```bash
# Using Render Shell (Dashboard → Service → Shell)
python manage.py createsuperuser
```

Or use the demo admin account:
- Username: `admin`
- Password: `admin123`

### 2. Update Frontend API URL

Update `REACT_APP_API_URL` in frontend environment variables to point to your deployed backend.

### 3. Update CORS Settings

Update `CORS_ALLOWED_ORIGINS` in backend to include your frontend URL.

## Render Free Tier Limitations

- **Spin Down:** Services sleep after 15min inactivity (50s cold start)
- **Database:** 90-day expiration (backup your data!)
- **Bandwidth:** 100GB/month
- **Build Minutes:** 500 minutes/month

💡 **Tip:** Keep services awake with [UptimeRobot](https://uptimerobot.com) pinging every 14 minutes.

## Troubleshooting

### Build Fails

```bash
# Check build logs in Render Dashboard
# Common issues:
# - Python version mismatch → Set PYTHON_VERSION=3.12.0
# - Missing dependencies → Check requirements.txt
```

### Database Connection Error

```bash
# Verify DATABASE_URL is set correctly
# Check database status in Render Dashboard
```

### CORS Errors

```bash
# Update CORS_ALLOWED_ORIGINS in backend
# Must match exact frontend URL (including https://)
```

### Static Files Not Loading

```bash
# Run in Render Shell:
python manage.py collectstatic --no-input
```

### Service Won't Start

```bash
# Check health endpoint:
curl https://your-backend.onrender.com/api/health/

# View logs in Render Dashboard
```

## Custom Domain (Optional)

### Backend

1. Dashboard → Service → Settings → Custom Domain
2. Add your domain (e.g., `api.yourdomain.com`)
3. Update DNS records as instructed
4. Update `ALLOWED_HOSTS` environment variable

### Frontend

1. Dashboard → Static Site → Settings → Custom Domain
2. Add your domain (e.g., `yourdomain.com`)
3. Update DNS records
4. Update `REACT_APP_API_URL` to point to custom backend domain

## Environment Variables Reference

### Required (Backend)

| Variable | Description | Example |
|----------|-------------|---------|
| `SECRET_KEY` | Django secret | Auto-generated |
| `DATABASE_URL` | PostgreSQL connection | Auto-generated |
| `ALLOWED_HOSTS` | Allowed domains | `civicfix-backend.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | Frontend URLs | `https://civicfix-frontend.onrender.com` |

### Optional (Backend)

| Variable | Description | Default |
|----------|-------------|---------|
| `DEBUG` | Debug mode | `False` |
| `EMAIL_HOST_USER` | SMTP username | - |
| `EMAIL_HOST_PASSWORD` | SMTP password | - |
| `SLA_HOURS` | Auto-escalation time | `24` |

### Required (Frontend)

| Variable | Description | Example |
|----------|-------------|---------|
| `REACT_APP_API_URL` | Backend API URL | `https://civicfix-backend.onrender.com/api` |
