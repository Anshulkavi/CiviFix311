### STEP 6: Format in Word

#### Heading Styles:
- **Chapter Titles:** Heading 1 (e.g., "CHAPTER 1: INTRODUCTION")
- **Section Numbers:** Heading 2 (e.g., "1.1 Background")
- **Subsections:** Heading 3 (e.g., "1.1.1 Historical Context")

#### Text Formatting:
- **Body Text:** Times New Roman or Arial, 12pt, 1.5 line spacing
- **Code Blocks:** Courier New or Consolas, 10pt, gray background
- **Tables:** Auto-fit, bordered, alternate row shading
- **Figures:** Center-aligned, caption below (e.g., "Figure 4.1: System Architecture")

#### Page Setup:
- **Margins:** 1 inch (2.54 cm) all sides
- **Page Numbers:** Bottom center, starting from Introduction (page 1)
- **Front Matter:** Roman numerals (i, ii, iii, iv...)
- **Main Content:** Arabic numerals (1, 2, 3...)

#### Table of Contents:
- Use Word's built-in TOC feature
- References → Table of Contents → Automatic Table
- Update before final submission (Right-click TOC → Update Field)

## 💡 TIPS FOR SUCCESS

### 1. Diagram Quality
- Export diagrams at **300 DPI** for print quality
- Use **PNG** format (not JPG) to avoid artifacts
- Ensure text in diagrams is **readable** (minimum 10pt font)
- Use **consistent colors** across all diagrams

### 2. Screenshots
- Take screenshots at **1920x1080** resolution
- Use **full-page** screenshots showing browser context
- Add **annotations** (arrows, labels) using image editor
- **Blur** any sensitive data (if using real data)

### 3. Code Blocks
- Keep code blocks **under 30 lines** for readability
- Add **comments** in code to explain logic
- Use **ellipsis (...)** to indicate omitted code
- Include only **essential** code, not entire files

### 4. Tables
- Use **zebra striping** (alternate row colors)
- Keep columns **uniform width** where possible
- **Bold** header rows
- Use **center alignment** for numeric data

## ❓ TROUBLESHOOTING

**Problem:** Mermaid diagram too complex, won't render
- **Solution:** Simplify, break into smaller diagrams, or use Draw.io

**Problem:** Screenshots too large, file size > 50 MB
- **Solution:** Compress images (Right-click in Word → Format Picture → Compress)

**Problem:** Code doesn't fit on one page
- **Solution:** Use smaller font (9pt), narrow margins, or split across pages

**Problem:** Table of Contents not updating
- **Solution:** Right-click TOC → Update Field → Update Entire Table

**Problem:** Page numbers wrong
- **Solution:** Insert → Page Break → Section Breaks, format sections separately

**Problem:** Diagrams look pixelated when printed
- **Solution:** Re-export at 300 DPI, use PNG format
